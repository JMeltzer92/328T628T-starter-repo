### Part 1

Go through each demarcated section of the `main.js` file in this branch. Uncomment the code, line-by-line or in chunks, and tell me how that chunk is working. Do not uncomment the section headers. For each numbered section, in the next box in ELMS, describe how the code works. Separate your thoughts by section. No need to write a novel, just give me the basics of how it's working. **Do this through at least section 5 - you may go further if you wish to 6 and 7.**

### Part 2 

#### Option A

Check out the hexagons on the [D3 landing page](https://d3js.org/) - they all link to different D3 prototypes or data-driven essay-visualizations in Observable (Mike Bostock's goated documentation and testing platform for D3). Check out these prototypes. Which ones do you like most? Which expand your definition of what's possible with data visualization or interactive graphics? Observable code has a slightly different syntax than JavaScript... but are there any functions or capabilities that you might want to explore further? Are there any data methodology questions you've come across that you want to discuss?

Basically, however you do it, this is a meditation on what D3 can do and how JavaScript libraries like it have come to power the modern web and the news data viz space. In the ELMS text box, write up your thoughts in short essay format. Aim for about 500 words.

#### Option B

Try making your own D3 chart, with your own data. You will probably want to use an AI coding assistant for this - make sure you go slow and step-by-step in your instructions. I recommend sticking to familiar chart types with easily structured data - think scatterplots and bar/line charts instead of Sankey diagrams or animated infographics. Your basic approach should be:

1. Load your data
2. Prepare your chart area
3. Prepare your scales - range and domain!
4. Draw your chart
5. Add furniture (head, subhead, source, credit), usually in HTML but can be done in your SVG as well

You can take this chart as far as you like (animation, annotations, interactivity...). 

Use this repository, overwriting my code with your own. It may be helpful to replace my code with yours as you go, as well.

You must submit:

- A github pages link to your live chart
- A github repo link to your code
- A very brief (couple sentences) description of your D3 chart

Pro tip - RawGraphs has sample data sets you can use, and that platform is essentially D3 code that spits out an SVG for you to edit. The sample datasets each have "suggested chart types" to go with them. Grab the sample data and try making the suggested chart type, but in D3.

Good luck to you if you choose this option! Remember, I am around to help with any code challenges you encounter.

If you want to make a pass at one of your final project deliverables and get feedback ahead of time, this is a great opportunity to do so. 

Be warned, you will need to turn in something polished for this option. None of these tools are new at this point, so you need to put a lot of thought and care into them, especially if you decide to make another Flourish graphic. It should be a new chart type you haven't submitted yet.

You must submit:
